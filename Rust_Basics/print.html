Rust Basics I

Introduction

Rust is a strong static language that brings rapid speed, memory safety and concurrency together. It's typically used in backend applications (web servers, micro services, command line applications) and is typically for system programming. In a more closer application sense, Rust is used as the main and preferred language for development in MKProjects!

Bindings and Mutability

Before we talk about what Rust calls bindings, and mutability, let's look at a simple rust program first:

hello-world.rs

fn main(){
    println!("Hello World!");
    // Press play to see output
}
  • Rust is similar to programming languages such as C or JavaScript with the syntax that it follows.
  • Most programming languages contains variables, and in Rust they are called bindings.
    • let var = "value;
  • By default, bindings are immutable (value cannot be changed)

If we try to set our binding var to another value, the code would not compile: 

fn main(){
    let var = "value";
    var = "something else";
    println!("{}", var);
    // Press play to see the compile error
}

However Rust does also support mutable bindings in the form of the keyword mut. We could try the program above again, but instead make the binding var mutable. 

fn main(){
    let mut var = "value";
    var = "something else";
    println!("{}", var);
    // Press play to see output
}

Unlike other programming languages, the Rust compiler will prevent uses of uninitialized bindings:


#![allow(unused)]
fn main() {
let mut var; //Uninitialized (set to no value)
do_something(var); // Results to error due to uninitialized binding
}

as they contain garbage values and lead to bugs in the code, so instead they are considered invalid.

Basic Types

Let's first look more into the anatomy of a simple rust program, we have the following:

  • fn : which denotes a function
  • main() : a main function is where code is executed by the compiler
  • {} : the function body is where the code is put in for the function

Rust's compiler can usually assume the type of a binding, such as let num = 3.0 could be a 64-bit floating-point number or f64. However, it is good practice to also let Rust know the type, and to do so, you need to know the different types that are in Rust.

Scalar Types

A scalar type represents a single value. Rust has four primary types, which are: integers (1, 2, 3), floating-point numbers (1.0, 2.1, 3.6), Boolean (true, false), and characters ('c', 'a').

To have a binding declare a type, just place a colon after it with the type: 

fn main(){
    let int_num : i32 = 45; // 32 bit integer
    println!("The integer is {}!", int_num);
}

Integers

An integer is a number in which has no fraction component associated with it, and can be only positive (unsigned), or positive/negative (signed) in various lengths of bits.

Note: Signed integers start with i while unsigned integers start with u.

Floating-Point Numbers

Floating-point numbers are numbers that do have a fractional component (or decimal). In Rust, there are only a 32-bit and 64-bit variants or f32 and f64 respectively. The default is f64 and it is considered to be double precision point, while f32 is considered to be a single precision point. 

fn main(){
    let float1 = 78.5; //By default is f64
    let float2: f32 = 87.6;
    println!("This is f64: {}, this is f32: {}", float1, float2);
}

Operations

Rust supports the basic arithmetic that you'd probably expect to do, addition, subtraction, multiplication, division, and modulo. The code below will show an example of each: 

fn main(){
    let sum = 6.7 + 9.8; // addition 

    let difference = 10 - 7; // subtraction

    let product = 78 * 43; // multiplication

    let quotient = 8.0/3.0; // division

    let remainder = 35 % 6; // modulo
}

Note: You cannot do operations between an integer and floating-point.

Boolean

As like most programming languages, a Boolean type has two possible values, true and false. These values are both one byte in size and a bool type is issued with the keyword bool. 

fn main(){
    let heads: bool = true; //explicit type annotation

    let tails = false; 
}
